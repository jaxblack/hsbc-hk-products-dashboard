# 真实交易依赖风险 / Real-Trading Dependency Risk

本仓库**不是交易系统、不是行情系统、也不是投顾系统**。本文件存在的
唯一目的，是把"如果有人想拿它驱动真实下单"会面临的风险列清楚，让
任何阅读者在动这个念头前就先打消。

> 👉 一句话：**不要**用本看板的数据驱动任何真实交易。如果你确实需要
> 真实交易数据，请直接登录 HSBC HK 官网 / app 或与你的客户经理确认。

## 为什么不能依赖

### 1. 数据来源是"展示性页面"，不是"成交性数据"

- HSBC HK 的公开 deposit-rate 页面是**对潜在客户的展示**，不是交易确认。
  实际客户在登录后获得的报价、可用余额段、起存金额、罚息条款都可能
  与公开页面**不一致**。
- 该页面是**人类阅读**导向，HSBC 不承担"机器读取后再消费"的准确性
  承诺。

### 2. 数据有显著延迟

- 抓取频率：本仓库建议为每日 1 次（参见 [data-latency.md](data-latency.md)）。
- 利率可以在工作日内变更（尤其是 USD 在 FOMC 公告日之后），公开页面
  与可成交利率可能在分钟级、小时级出现错位。
- 即便 cron 提高到每小时 1 次，**依然不是实时**；银行后台调价、页面
  缓存刷新都各有节奏。

### 3. 数据条目不是合同条款

- 字段 `balance_band` 是"档位描述"，不是"起存金额"；起存金额可能更
  高（例如某档显示 `HKD500,000 and above` 不代表 500,000 就一定能开，
  实际还受最低开户金额、币种最小单位约束）。
- `risk_level` / `fee` 字段在本仓库当前为 HSBC 页面原文或静态默认值；
  **不代表**完整的费用结构（手续费、提前支取罚息、汇兑差价等都未
  覆盖）。
- 任何 footnote、星号 `*`、"detailed terms apply" 文本在解析阶段
  **可能被丢弃**；这些 footnote 往往才是真正具有约束力的合同条款。

### 4. 解析层的脆弱性

- scraper 用 `bs4` 对 HTML 做 best-effort 解析；HSBC 改版 / A/B 测试
  可能导致：
  - 某些条目静默丢失（看板少了一行，但不报错）；
  - 利率数值被错位（例如某栏整列右移一格）；
  - 单位 / 货币被误识别（HK 页面的 `RMB` 与 `CNY` 混用历史已经踩过坑）。
- 我们做了 [§ Invariants](metrics.md#数据质量约束--invariants) 一类基础校验，
  但**不**能保证"数值都对"——只能保证"数值是合法的浮点数 / 在合理区间"。

### 5. 没有任何成交链路

- 本仓库**不**对接任何下单接口；技术上也无法替你"按这个利率开一笔
  定存"。
- 本仓库**不**保存你的身份、账户、持仓；没有任何"代客确认"能力。
- 即便有第三方写一个外挂去把本看板的输出灌进交易接口——他们必须
  自己承担："数据是延迟的、是 best-effort 解析的、是非官方的"这三
  条已知风险，否则一定会出问题。

## 如果你必须把它接入下游"半自动"工具

下面是**最低限度**的护栏（不构成合规背书，仅供工程参考）：

| 护栏 | 强度 | 说明 |
|------|------|------|
| 在下游每次使用前，二次拉取官方页面校对 | 必需 | 用本仓库选目标 → 然后**人工**在 HSBC 官网核对该利率才行动。 |
| 限定"仅作筛选 / 排序"，不参与金额 / 时点决策 | 必需 | 利率选好后，**金额 / 时点 / 期限**由人来定。 |
| 在下游 UI 显眼位置重复免责声明 | 必需 | 不能只在本仓库的页面上声明就完事；下游也要再说一遍。 |
| 任何告警 / 推送都不写"建议买入 / 卖出 / 申购" | 必需 | 只写客观事实，参见 [alerts.md § 红线](alerts.md#关于投资告警的红线)。 |
| 不留任何客户标识 / 持仓 / PII | 必需 | 本看板设计为"无用户态"；下游若加入用户态须独立合规 review。 |

## 我们的立场

- 本仓库的维护者**不**对任何依赖本看板做出的决策负责。
- 如发现本仓库被用于"自动化交易 / 代客建议 / 误导性产品推荐"，欢迎
  在 issue 中反馈，我们会进一步加强免责声明和访问限制。
